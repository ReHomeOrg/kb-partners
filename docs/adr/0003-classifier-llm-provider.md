# ADR-0003 — Боевой LLM-провайдер классификатора (YandexGPT)

- Статус: Принято (решение Архитектора, 2026-06-11)
- Дата: 2026-06-11
- Контекст: ТЗ `kb-partners` v1.1 §4.11, E2 (FR-2.1–2.6), §16.9, §16.13; CLAUDE.md правило 6

## Контекст

Классификатор (E2) определяет категорию заявки. Быстрый путь — детерминированные
правила (ключевые слова); при неоднозначности/отсутствии совпадений нужен LLM
(FR-2.3). До этого ADR LLM-путь был инертен (`NullLLMProvider`, rules-only).
§16.9 оставлял открытым выбор провайдера и режим.

Ограничения: все ПДн-обработки — в РФ (ФЗ-152); правило 6 запрещает новые внешние
SDK без ADR; принцип «разрабатываем сами».

## Решение

**Боевой провайдер по умолчанию — YandexGPT** (Yandex Cloud Foundation Models),
РФ-контур, ФЗ-152-совместим. Подключение — **собственным HTTP-адаптером** на
`ResilientHttpClient` (httpx), **без вендорского SDK**; Api-Key-авторизация;
переключение по env (`classifier_llm_provider=yandexgpt`).

- Реализация: `api/classifier/yandexgpt.py::YandexGptProvider` (Protocol `LLMProvider`).
- Эндпоинт: `POST {yandexgpt_api_base_url}/foundationModels/v1/completion`,
  `modelUri=gpt://<folder>/<model>/latest`, `temperature=0`.
- Config-gated: пустые `yandexgpt_api_key`/`yandexgpt_folder_id` → `NullLLMProvider`
  (rules-only), сервис не падает. Секреты — ссылкой на kb-vault, не инлайн.
- **ФЗ-152:** на вход модели идёт ТОЛЬКО `raw_input_masked` (ПДн вырезаны при приёме,
  FR-1.6); текст промпта/ответа в логи не пишется. Модель просим вернуть строгий JSON;
  неразбираемый ответ/недоступность/4xx → `None` → деградация на rules-путь/NEEDS_REVIEW.
- Трассируемость (FR-2.6): `model`/`version` сохраняются в `classification`.

Альтернативы: **GigaChat (Сбер)** — равноценная РФ-альтернатива, может быть включена
по тому же контракту отдельной реализацией; **self-hosted vLLM** — максимум «сами»,
но требует инфраструктуры. Оставлены как зарезервированные имена провайдера.

### Качество (golden-eval, §16.13)

Стартовый golden-набор — `backend/tests/golden/classifier_golden.json` (маскированные
тексты, без ПДн). Гейт: точность **rules-пути** на наборе ≥ **0.8** (дефолт, уточняется
Архитектором). Владелец/расширение набора и целевой порог — Архитектор/kb-eval.

## Последствия

- LLM-классификация доступна боевым включением env; по умолчанию — инертна (rules-only).
- Смена провайдера/модели = правка адаптера + env, без изменения сервисного слоя.
- Реальная стоимость/латентность LLM — на ops-мониторинге (метрики `external_client_*`).
